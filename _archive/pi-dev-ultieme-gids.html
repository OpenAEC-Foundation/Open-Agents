Pi.dev Ultieme Gids

Van Claude Code gebruiker naar Pi power user

Impertio Studio B.V. · 26-02-2026 · Provider: Anthropic (Claude)

Pi is een minimale, open-source terminal coding agent die jou volledige controle geeft over wat er in het context window zit. Anders dan Claude Code's feature-rijke aanpak, geeft Pi je 4 tools en een 200-token systeemprompt — en laat je de rest zelf bouwen via TypeScript extensions, skills en packages.

Installatie

1
Node.js installeren (als je dat nog niet hebt)
Pi vereist Node.js 18+. Check met node --version
2
Pi globaal installeren
Via npm — één commando
npm install -g @mariozechner/pi-coding-agent
3
Verifiëren
Check of pi beschikbaar is
pi --version
pi --help
Op Windows heb je bash nodig. Pi zoekt automatisch naar Git Bash op C:\Program Files\Git\bin\bash.exe. Als je al Claude Code in PowerShell gebruikt, heb je waarschijnlijk Git Bash al. Check anders je PATH voor bash.exe (Cygwin, MSYS2, WSL).

Anthropic/Claude Configureren

Maak het authenticatie bestand:

~/.pi/agent/auth.json
{
  "anthropic": {
    "type": "api_key",
    "key": "sk-ant-api03-jouw-key-hier"
  }
}

Drie manieren om je key op te geven

MethodeVoorbeeldWanneer
Letterlijke key"sk-ant-api03-..."Simpel, lokale dev
Shell commando"!op read 'op://vault/anthropic'"1Password, macOS Keychain
Env variabele"ANTHROPIC_API_KEY"CI/CD, containers
In Claude Code wordt authenticatie afgehandeld via claude login. In Pi configureer je het zelf via auth.json of /login voor OAuth (Claude Pro/Max abonnement).

Eerste Sessie

# Nieuwe interactieve sessie
pi

# Met initiële prompt
pi "Leg deze codebase uit"

# Specifiek model kiezen
pi --model claude-sonnet-4-20250514 "Refactor dit bestand"

# Model met thinking level
pi --model sonnet:high "Los dit complexe probleem op"

Bij de eerste start detecteert Pi automatisch je terminal achtergrond (dark/light) voor het thema. De startup header toont geladen AGENTS.md bestanden, prompt templates, skills, en extensions.

De Terminal Interface

Pi is een scrollback-based TUI. Niet full-screen zoals vim, maar het voegt toe aan je terminal buffer met native scrolling en zoeken. Dit werkt het beste in Ghostty en iTerm2.

Interface zones

ZoneWaarWat
Startup headerBovenaanShortcut hints, geladen context bestanden
Messages areaMiddenConversatie, tool calls, outputs, errors
EditorOnderaanHier typ je. Randkleur = thinking level
FooterOnderaanDir, sessie, tokens, kosten ($), context %, model
Typ @ voor fuzzy file search (respecteert .gitignore). Tab completeert bestandspaden. Je kunt bestanden drag & droppen vanuit je file manager naar de terminal.

Steering vs Follow-up

Dit is het grootste verschil met Claude Code. In Claude Code wacht je altijd tot de agent klaar is. In Pi kun je actief sturen terwijl de agent werkt.
Enter — SteeringAlt+Enter — Follow-up
Wanneer?Tijdens streamingTijdens streaming
GedragGeleverd na huidige tool, onderbreekt restWacht tot agent alles afrondt
GebruikKoerscorrectie: "Stop, doe X anders"Vervolgvraag: "Daarna ook Y doen"

Interactive Mode (standaard)

pi                              # Nieuwe sessie
pi "Refactor auth module"       # Met prompt
pi -c                           # Hervat meest recente
pi -r                           # Browse eerdere sessies
pi --session pad.jsonl          # Specifieke sessie

Print / JSON Mode

Niet-interactief, perfect voor scripts en CI/CD:

pi -p "Vat deze codebase samen"              # Tekst naar stdout
pi --mode json "Leg uit"                      # JSON event stream
pi --tools read,grep,find,ls -p "Review"     # Read-only mode

RPC Mode

JSON protocol over stdin/stdout — voor IDE integratie of non-Node.js apps:

pi --mode rpc

Commando's: prompt, steer, follow_up, abort, get_state, get_stats, select_model, list_models, bash, en meer.

SDK Mode

Programmatische embedding — dit is de modus voor ERPNext integratie:

import { AuthStorage, createAgentSession, ModelRegistry, SessionManager }
  from "@mariozechner/pi-coding-agent";

const authStorage = AuthStorage.create();
const modelRegistry = new ModelRegistry(authStorage);

const { session } = await createAgentSession({
  sessionManager: SessionManager.inMemory(),
  authStorage, modelRegistry,
});

session.subscribe((event) => {
  if (event.type === "message_update" &&
      event.assistantMessageEvent.type === "text_delta") {
    process.stdout.write(event.assistantMessageEvent.delta);
  }
});

await session.prompt("Welke bestanden staan hier?");

Sessies & Boom-structuur

Sessies zijn JSONL bestanden met een boomstructuur (id + parentId). Alle branches blijven bewaard in één bestand. /tree laat je navigeren naar elk punt en branches wisselen.

# Sessies beheren
pi -c              # Hervat meest recente
pi -r              # Browse interactief
pi --no-session    # Ephemeral — niets opslaan

# In sessie
/tree              # Boom navigeren, branches wisselen
/fork              # Nieuwe sessie vanuit huidige punt
/export            # Exporteer naar HTML
/share             # Upload naar GitHub gist

AGENTS.md — Project Instructies

Geladen bij startup vanuit drie locaties (geconcateneerd):

~/.pi/agent/AGENTS.md    # Globaal
../AGENTS.md             # Parent dirs tot git root
./AGENTS.md              # Huidige werkdirectory
Schrijf een AGENTS.md voor elk ERPNext project met API endpoints, DocTypes, filter syntax, en regels. Dit is Pi's equivalent van Claude Code's CLAUDE.md.

Compactie

Automatisch standaard ingeschakeld. Vat oudere berichten samen bij context overflow. Configureerbaar:

/compact                         # Handmatig
/compact behoud alle API calls   # Met instructies

# Settings
compaction.reserveTokens: 16384       # Tokens voor LLM respons
compaction.keepRecentTokens: 20000    # Recent berichten intact

Skills

On-demand capability packages. Alleen beschrijvingen zitten in context — instructies laden wanneer nodig.

# Skill directory structuur
mijn-skill/
├── SKILL.md            # Vereist: YAML frontmatter + instructies
├── scripts/
│   └── process.sh      # Helper scripts
└── references/
    └── api-ref.md      # Docs die on-demand laden
~/.pi/agent/skills/erpnext-todos/SKILL.md
---
name: erpnext-todos
description: Manage ERPNext ToDo items via Frappe REST API.
  Create, list, update, and close todos.
---

# ERPNext ToDo Management

## List open todos
```bash
curl -H "Authorization: token KEY:SECRET" \
  "https://instance.frappe.cloud/api/resource/ToDo?\
  filters=[[\"status\",\"=\",\"Open\"]]"
```

## Create todo
```bash
curl -X POST "https://instance.frappe.cloud/api/resource/ToDo" \
  -H "Authorization: token KEY:SECRET" \
  -d '{"allocated_to":"user@mail","description":"..."}'
```

Discovery locaties: ~/.pi/agent/skills/, ~/.agents/skills/, .pi/skills/, .agents/skills/, en packages.

Prompt Templates

Elk .md bestand in de prompts directory wordt een slash command:

~/.pi/agent/prompts/review.md
Review deze code op bugs, security issues, en performance.
Focus op: {{focus}}

Typ /review → Pi expandeert naar de inhoud. Ondersteunt {{variable}} en bash-argumenten ($1, $@).

Extensions

TypeScript modules met volledige systeemtoegang — tools, commands, shortcuts, CLI flags, UI, en 20+ events.

~/.pi/agent/extensions/my-ext.ts
import type { ExtensionAPI } from "@mariozechner/pi-coding-agent";

export default function (pi: ExtensionAPI) {
  // Custom tool voor het LLM
  pi.registerTool({
    name: "erpnext_query",
    label: "ERPNext Query",
    description: "Query ERPNext via REST API",
    parameters: Type.Object({
      doctype: Type.String(),
      filters: Type.Optional(Type.String()),
    }),
    async execute(id, params, onUpdate, ctx, signal) {
      const resp = await fetch(
        `https://instance.frappe.cloud/api/resource/${params.doctype}`,
        { headers: { "Authorization": "token key:secret" } }
      );
      const data = await resp.json();
      return {
        content: [{ type: "text",
          text: JSON.stringify(data, null, 2) }],
        details: {},
      };
    },
  });

  // Slash command
  pi.registerCommand("todos", {
    description: "Toon open ERPNext todos",
    handler: async (args, ctx) => {
      ctx.ui.notify("Todos laden...", "info");
    },
  });
}

Wat moet je afleren van Claude Code?

1. Stop met verwachten van ingebouwde safety prompts — bouw een permission-gate extension
2. Stop met MCP servers — schrijf CLI tools met README bestanden
3. Gebruik tmux voor background processen
4. Schrijf plannen naar bestanden (PLAN.md) i.p.v. een plan mode
5. Bouw CLI tools eerst, wrap ze optioneel in extensions

ERPNext/Frappe Integratie

Architectuur: SDK als Sidecar

ERPNext (Python)
    ↕ HTTP
Pi Agent Service (Node.js + Pi SDK)
    └─ Custom tools:
       ├─ frappe_get_document
       ├─ frappe_list_documents
       ├─ frappe_create_document
       ├─ frappe_call_method
       └─ frappe_run_report
    └─ GEEN bash/write/edit tools
In productie: nooit de standaard bash/write/edit tools aanbieden aan de agent. Maak custom tools die alleen via de Frappe REST API werken. Zo kan de agent nooit buiten zijn scope opereren.

Security

Pi levert bewust geen ingebouwde security. Praktische opties:

MaatregelHoe
Permission gatesExtension die tool_call events intercepteert
Protected pathsExtension die file ops op specifieke dirs blokkeert
SandboxingExtension die commands in Docker/Podman draait
Read-only--tools read,grep,find,ls flag
API isolatieDedicated Frappe users met minimale roles
Audit trailLog tool calls naar custom DocType
